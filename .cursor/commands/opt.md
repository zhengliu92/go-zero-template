# opt - 代码逻辑优化检查

分析当前选中的代码或文件，检查是否存在可优化的逻辑，如果可优化则执行优化。

## 检查项目

1. **性能优化**
   - 不必要的循环或重复计算
   - 可以提前返回的条件判断
   - 可以并行处理的操作
   - 数据库查询优化（N+1 问题、缺少索引使用）

2. **代码简化**
   - 冗余的变量或代码块
   - 可以合并的相似逻辑
   - 过于复杂的条件判断
   - 可以使用更简洁的语法

3. **错误处理**
   - 缺失的错误检查
   - 可以提前处理的边界情况
   - 错误信息是否清晰

4. **可读性**
   - 变量/函数命名是否清晰
   - 是否需要添加注释
   - 代码结构是否合理

## 执行流程

1. 阅读并分析选中的代码
2. 列出发现的可优化点（如有）
3. 如果存在可优化项，直接执行优化修改
4. 简要说明优化内容和原因

## 注意事项

- 保持代码功能不变
- 遵循项目现有的代码风格
- 优先考虑可读性和可维护性
